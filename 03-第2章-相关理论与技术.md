第 2 章 相关理论与技术

2.1 SpringBoot框架介绍

Spring Boot 是 Spring 生态的轻量化封装，强调“约定优于配置”，通过自动配置与 starter 机制快速集成 Web 与数据访问能力，减少 XML 与繁琐配置带来的开发成本。本系统后端基于 Spring Boot 3.2.8 与 Java 17 构建，使用 spring-boot-starter-web 提供 REST 接口能力，结合 spring-boot-starter-data-jdbc 统一数据访问入口，并通过内嵌容器保证服务可独立运行。业务层采用 Controller、Service、Mapper 的分层组织方式，控制器承接管理员登录、班级与课程维护、开课计划与课表查询、教学事故与通知公告等请求，Service 层集中处理业务校验与流程编排，Mapper 层对接 MyBatis-Plus 完成持久化操作；同时借助 springdoc-openapi 与 Knife4j 输出接口文档，便于接口对接与联调，从而在保持结构清晰的同时提升教务业务的迭代效率与可维护性。

2.2 Vue.js前端框架

Vue.js 是渐进式 JavaScript 前端框架，核心特征包括组件化开发、响应式数据驱动与单文件组件（SFC），适合构建交互频繁的单页面应用（SPA），并能在保持结构简洁的同时提升页面复用与渲染效率。

在教务管理系统的界面层设计中，Vue.js 适合将课程、班级、教师、开课计划、课表、教学事故、通知公告等页面拆分为独立组件，并通过路由机制组织导航层级，保持角色视图与功能入口的清晰一致；配合状态管理保存登录态、检索条件与分页参数，可避免跨页面数据丢失；与 Axios 的统一请求封装结合后，能够将前端表格、表单与筛选操作与后端 REST 接口稳定衔接，必要时可结合 Element UI 等组件库提升表格与表单的交互体验，从而满足教务场景下高频查询与数据录入的前端呈现需求。

2.3 MyBatis-Plus数据访问框架

MyBatis-Plus 是对 MyBatis 的增强框架，在保留手写 SQL 与灵活映射能力的基础上，提供通用 CRUD、分页与条件构造等高频能力，降低数据访问层的重复编码成本。其 BaseMapper 接口与条件构造器可简化常见查询，提升数据层的可读性与扩展性，适合教务系统中多表、多条件的检索需求。

系统在 Mapper 层统一继承 BaseMapper，课程、班级、教师、公告、开课计划与课表等数据的增删改查由框架自动生成；在业务查询中使用 QueryWrapper 与 LambdaQueryWrapper 进行条件拼装，配合 Page 完成分页返回，满足课程列表、教师列表与公告分页等场景；实体类中通过 @TableField 标注非表字段，保证视图扩展与持久化字段的边界清晰。同时，application.properties 中开启下划线转驼峰映射，使数据库字段与实体属性保持一致，提升模型与 SQL 的一致性与可维护性。

2.4 MySQL数据库

MySQL 是成熟的关系型数据库，支持事务与多表关联，适合结构化业务数据的存储与一致性控制。本系统以 MySQL 作为核心数据存储，通过 JDBC 驱动与连接池完成数据接入，数据库名为 edums，集中存放管理员、班级、课程、开课计划、课表、教师、教学事故、系统配置与公告等核心表结构，保证教务数据可追溯与业务流程可闭环。课程开设与课表安排需要同时关联课程、教师与班级信息，教学事故与公告管理又涉及跨表查询与统计，MySQL 的关系模型能够提供可靠的数据约束基础，为排课查询、公告发布与事故记录等功能提供稳定的数据支撑。

2.5 Java语言

Java 具备跨平台、面向对象与完善生态等特点，适合作为企业级后端开发的核心语言。本系统使用 Java 17 作为开发语言，通过类与接口抽象业务实体与服务边界，配合注解式开发与依赖注入保证模块解耦；实体层利用 Lombok 的 @Data 注解减少样板代码，使课程、班级、教师、公告等实体字段定义更清晰；控制层与服务层通过异常处理与参数校验保障接口稳定性，并结合集合框架对列表、分页结果进行组织与封装。Java 的类型系统与工程化能力为后端业务逻辑、接口组织与数据处理提供了稳定基础，也为后续功能扩展与维护提供了良好的可持续性。
