摘  要

在高等教育数字化治理与教学质量评价常态化的背景下，高校教务业务覆盖班级、课程、开课计划、课表与教学事故等多环节，但现有流程存在数据分散、业务口径不一、排课与查询依赖多系统、事故处理缺少闭环等问题，导致教师与管理人员难以在同一平台完成计划编排、查询统计与通知发布，教学数据的可追溯性与监管力度不足。针对上述问题，本文基于 Java 17 与 Spring Boot 3.2.8 构建教务管理系统后端，采用 MyBatis-Plus 与 MySQL 形成统一数据模型与分页查询机制，并围绕管理员登录与教学配置、教师注册与信息维护、班级与课程管理、开课计划与课表安排、教学事故记录、通知公告管理及首页统计等功能建立业务闭环，配套 OpenAPI/Swagger 接口文档支撑系统集成与扩展，从而提升数据一致性、排课与查询效率、过程追踪能力与管理决策的及时性，为高校教务流程规范化和教学质量治理提供技术支撑。

关键词：教务管理；课程安排；Spring Boot；MySQL
